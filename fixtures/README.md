# Fixture Data Policy

Fixture payloads are intentionally not tracked in Git. They include reduced real-data exports that are useful for local
integration tests, but some files are tens of megabytes and generated test outputs duplicate them many times.

Use `make fixtures` from the `integration_test/` directory to regenerate the standard MaxQuant, MSstats, FP_TMT, and
FP_singlesite fixtures from the source packages and Zenodo archive listed in the main README. The WU345302 DIA-NN
fixture is not regenerated by that target, so `make clean` intentionally preserves `fixtures/diann_wu345302`.

The WU345302 DIA-NN fixture should stay out of normal Git history unless the project deliberately adopts Git LFS or an
external release/download mechanism for it.

Commit only lightweight fixture metadata when needed:

- small config templates
- dataset annotations
- checksums or manifests
- instructions for obtaining or regenerating data

Do not commit generated reports, logs, parquet/RDS/XLSX outputs, copied input payloads, or large raw exports.
